# ตัวอย่าง: Raw Data — กระบวนการจัดซื้อทรัพย์สิน

## รูปแบบข้อมูลที่ User ให้ (ทยอยให้ทีละชุด)

### Batch 1: ขั้นตอนหลัก
```
จัดซื้อทรัพย์สิน
1. user ขอซื้อ
2. ถ้าซื้อแทนของเดิมมีใบspoil (ตัดจำหน่าย) ถ้าไม่มีต้องออกใบmemo
3. จากนั้น purchase จะสืบราคา...
...
10. ส่งให้ vendor
```

### Batch 2: ตัวอย่างประกอบ
```
เช่น ซื้อเตาย่างเสต็ก ยี่ห้อใหม่แทนของเดิมที่เสีย
Chef ส่งรูปมาให้ ข้อมูลร้านมาให้ --> purchase ไปเชคราคา...
```

### Batch 3: ขั้นตอนย่อย
```
การ screen vendor
1. ส่วนใหญ่ซื้อ vendor เก่า...
2. ถ้าเป็น vendor ใหม่ ขอ ภพ20

การสืบราคา
Vendor ส่งราคา marketing ให้เราทุก 15 วัน...
```

### Batch 4+: ประเด็นควบคุมภายใน
```
- ระบบ PR แสดงยอด balance คงเหลือ แต่ของจริงถูกใช้ไปแล้ว
- Outstanding PO ไม่มีการติดตาม
- PO ที่อนุมัติแล้วสามารถแก้ไขได้โดยไม่ต้อง re-approve
```

## Pattern ที่พบ

1. **User ให้ข้อมูลแบบ narrative** — ไม่ได้ตอบคำถามทีละข้อตาม template แต่เล่าเรื่องต่อเนื่อง
2. **User ไม่ใช้ `<>` เอง** — ต้องให้ agent ถาม probing questions ทีหลัง
3. **ประเด็นควบคุมภายใน** มักมาเป็น batch หลังๆ หลังจากให้ขั้นตอนหลักครบ
4. **มี typos/common shorthand**: "รร" = โรงแรม, "จนท" = เจ้าหน้าที่, "พิจารรา" = พิจารณา
5. **User ชอบให้ข้อมูลแบบ "ทยอย"** — ต้องบันทึก raw data ทันทีทุก batch

## คำถามที่ควรถามหลังเก็บข้อมูลครบ

- ชื่อระบบ ERP?
- DOA / วงเงินอนุมัติระดับต่างๆ?
- นโยบาย/ระเบียบที่เป็นลายลักษณ์อักษร?
- Physical controls (กล้องวงจรปิด, การจำกัดการเข้าถึง)?
- จุดที่ user ไม่แน่ใจ (เพื่อใส่ `<>`)?
